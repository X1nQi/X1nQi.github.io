发表于 2023-02-09 分类于 项目总结

基础功能的TodoList案例：具体的需求和分析

项目地址：TodoList项目

需求

  1. 输入待办事项后按回车添加至待完成列表中
    1. 在全局按下回车就可以添加
    2. 在内容为空时不添加
  2. 点击复选框可以切换事项的状态
  3. 双击代办事项内容可以进行修改
    1. 对于已完成的事项，不支持修改内容
    2. 失去焦点后确定修改
  4. 每个事项后面都有删除按钮，点击后删除

采用技术

  1. HTML5
  2. Sass
  3. bootstrap
  4. JavaScript
  5. parcel
  6. node

主要实现过程

在HTML中建立两个 ul标签，一个是 todolist 一个是 donelist，定义一个对象 list，存放 donelist 和 todolist，在初始化的时候，给需要使用的元素添加事件监听，最后判断localStorage内有没有list属性，如果有，就把本地存储中的 list 解析后赋值给对象 list，就渲染，渲染函数为 render();

render函数

调用 render 之后，会把 list 给覆写到 localStorage 中，使用for in进行遍历。for in遍历的是属性名，因为 list 的属性 有 donelist 和 todolist，而 donelist 和 todolist 是获取到的 ul 节点，这两个变量使用 var 定义，如果使用 let 在 window 上无法被访问到。

使用reduce把相对应数组的字符串进行相加，最终得到了完整的 HTML 片段，注意HTML的排列

在第一个 input 标签中，通过判断是否为 donelist 中的元素来动态添加 checked 属性

难点主要在于切换和删除，统一使用一个函数：switchList(targer,bool,remove)进行控制

设置两个变量存放列表，如果点击了一个未完成的事项，那么此时 input 的 checked 状态为 true，然后再进行传参，所以此时 arr 的值为 list.todolist，arr1的值为 list.donelist；

然后通过第三个参数 remove 进行判断：

​ 如果是 undefined，就获取点击元素的下一个同级元素，即含有事项内容的 p 标签

​ 如果不是 undefined，就获取点击元素的上一个同级元素，即含有事项内容的 p 标签

然后通过 indexOf 方法，匹配事项中的内容，返回这个事项的索引，再使用splice方法进行数组的删除，splice方法会改变原数组，并且返回一个数组，从获得的索引开始，删除一个属性，这个属性即我们的点击事项的内容。

再通过 remove 参数判断是否进行删除还是进行数组的变化，如果进行删除，就直接进行渲染，如果不进行删除，就把获取到的被删除的属性push到另外一个列表中去，再进行渲染

分析运行过程

浏览器请求资源，当全部资源获取未超时的情况下

  1. 调用浏览器的渲染进程开启渲染进程开始构建 DOM树，同时开一个线程用来构建CSSOM
  2. 当渲染进程遇到了 script 标签后，切换到 JS解析进程，开始解析，在源码中是引入了 bootstrap，这里就不展开说了，JS解析完成后，继续遍历 DOM生成DOM树
  3. 遇到了自己写的 script 标签，切换到 JS解析进程，先把源码通过解析器进行词法分析和语法分析生成AST，基准解释器 Ignition通过AST生成字节码，然后清楚AST，直接运行字节码；
  4. 先在栈底压入全局执行上下文，在执行前，将全局中的 var 和 function 声明获取，记录在全局对象上（浏览器为window），获取全局的let、const、class声明，并记录在 全局 scope 上，然后检查scope中的重复声明和 scope 和全局对象上的重复声明，有重复声明就直接报错，这是在执行前就会报错，检查完成后，对var变量赋初值 undefined，对function赋初值 为函数对象，let、const、class没有初值，注意，这都是执行前做的操作。
  5. 在 JS解析完成后，回到渲染进程，合并DOM和CSSOM生产渲染树，然后遍历渲染树进行布局，生成DOM的位置信息，然后再进行分层，这是为了提高渲染效率，然后就进行绘制，绘制的命令发出后，会在另一个线程——合成线程中进行，第一步先进行分块， 在分块时会开启线程池加速分块，分块是把界面分成一个个块，当视图发生变化时，可以提高渲染的效率。第二步进行光栅化，光栅化也会开启一个线程池并调用GPU进程来加速光栅化，这一步是为了生成像素信息，第三步 DRAW QUAN，这一步会打包指令并发送给GPU进程，GPU根据指令在屏幕上显示图像。

用到的重要API

  1. parentElement：获取父节点
  2. firstElementChild：获取第一个子节点，可以配合第一个 API 进行使用
  3. nextElementSibling：基于自身节点的父节点，获取自身节点的下一个同级节点，如果自身就是最后的节点，返回 null
  4. previousElementSibling：基于自身节点的父节点，获取自身节点的上一个同级节点，如果自身就是最后的节点，返回 null
  5. splice():接受两个参数，第一个参数是索引，第二个参数是删除的位数，改变原数组，返回值为数组
  6. indexOf():接受任意参数，根据参数对数组进行匹配，会返回第一个匹配值的索引
